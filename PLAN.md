# Implementation Plan

## Goal

Build a single-project ASP.NET Core app that:

1. Serves a webpage with an xterm.js terminal driven by a WebSocket.
2. Runs an Ink-scripted narrative game inside that terminal, with clickable choices.
3. Hot-reloads the Ink whenever a source file changes, preserving each live player's state best-effort.
4. Exposes an optional Monaco editor pane (gated by `?burger=true`) for in-browser knot editing.

## Design decisions (all confirmed with the user)

| Question | Decision |
|---|---|
| .NET version | .NET 10 |
| Ink compilation | Shell out to `inklecate.exe` (checked in at `tools/`, v1.2.1) |
| Ink runtime | `ink-engine-runtime.dll` bundled with inklecate v1.2.1, referenced directly by csproj |
| Clickable choices | OSC 8 hyperlinks with `ink://choice/{i}` scheme |
| State on hot reload | Best-effort `LoadState` on the new story; warn and keep prior story on failure |
| Sessions | Multi-user, per-connection, **ephemeral** (no persistence, no resume) |
| RESTART | Not a button — reloading the page = new WS = fresh story |
| Tag styling | Small ANSI vocabulary: `#color:*`, `#bold`, `#dim`, `#clear` |
| Source layout | Globals in `Story/globals.ink`; one knot per file under `Story/knots/**`; subfolders allowed |
| main.ink | Generated by an assembler into `Story/.build/main.ink` (never hand-edited) |
| Entry point | Knot named `start` |
| Missing-knot behavior | Auto-create stub `=== name ===\nTODO\n-> END` on first visit |
| Editor delivery | Monaco from CDN, in a resizable bottom split pane |
| Editor gate | `?burger=true` on URL (client UI) AND `burger` cookie + `IsDevelopment()` (server PUT) |
| Editor smarts (v1) | Highlighting + divert autocomplete + Ctrl+click jump-to-knot |
| Save trigger | Ctrl+S only |
| Knot-leaving prompt | Modal: Save / Don't Save / Always Save / Never Save. "Always/Never" remembered for page session only |
| "Don't Save" | Discards the buffer |
| Same-knot choices | No prompt |
| Tab close / reload | Standard browser `beforeunload` warning when dirty |
| Frontend toolchain | None — CDN script tags, plain JS |

## Wire protocol

JSON over WebSocket. Server → client:

- `{type:"text", ansi:"…"}` — narrative line
- `{type:"choices", items:[{i:0, text:"…"}, …]}` — present choices
- `{type:"knot", name:"…", source:"…"}` — active knot changed (broadcast on transition)
- `{type:"warn", ansi:"…"}` — out-of-band notification (e.g. reload failed)
- `{type:"end"}` — story reached `-> END`

Client → server:

- `{type:"choose", i:2}` — player picked choice index 2

## REST API

- `GET  /api/knots`         → `["start", "town_square", "tavern", …]` (for autocomplete + jump)
- `GET  /api/knot/{name}`   → raw file contents
- `PUT  /api/knot/{name}`   → write file. Requires `IsDevelopment()` AND `burger` cookie. 403 otherwise.

## Key invariants

- `Story/.build/` is purely derived — gitignored and disposable.
- Filename of each knot file must match its `=== name ===`. The assembler warns on mismatch and uses the in-file declaration as authoritative.
- A failed hot reload never breaks an active session — the prior compiled story keeps running until the next successful compile.
- File-write endpoint is locked behind two independent gates so a single misconfiguration can't expose it.

## Out of scope (for v1)

- Persistence of state across reconnects (would require cookies + a session store).
- Multi-author collaboration / conflict resolution on edits.
- Editing the globals file from Monaco (edit on disk for now).
- Inklecate-style live diagnostics surfaced into Monaco (errors only show up in the terminal as warnings).
- Auth — the editor is for the local author only.

## Open items to revisit after v1

- Per-knot buffers in Monaco (currently single buffer; "Don't Save" discards).
- Surface inklecate compile errors into Monaco directly (with squigglies).
- Optional CSS-styled choice rendering in xterm (vs current OSC 8 + ANSI bold/color).
- Whether to persist `Always Save` / `Never Save` across reloads (currently page-session only).
